# Codex Gospel: Research Audit

This skill prevents confident scientific nonsense. It forces live evidence, explicit uncertainty, numeric proof, and durable research memory.

## Current-State First

Before recommending:

1. Inspect current files, data, branch state, reports, or run artifacts.
2. Check whether the work is already done elsewhere in the repo.
3. Anchor important claims to commands, paths, line numbers, artifacts, or datasets when practical.

## Audit Shape

For each material gap:

- Expected: what the contract, hypothesis, or standard requires.
- Actual: what the current evidence shows.
- Gap: the difference.
- Priority: P0 blocker, P1 near-term risk, P2 medium-term issue, P3 polish.
- Confidence: high, medium, or low with basis.

Include a strengths section for serious audits.

## Numeric Claims

Do not recommend a value unless you can cite one of:

- Measurement from the current system or dataset.
- Theoretical bound or derivation.
- Credible same-domain paper, standard, or textbook.

If none exists, say the value is unpinned and propose a measurement path.

## Research Memory

When an idea, decision, pivot, or negative result has converged, record it in durable project docs. A good research-tree node says why the question matters, what was decided, what evidence supports it, and what would change the decision later.

## Closeout

Lead with the practical conclusion. Then give the evidence that changes the decision, the uncertainty that remains, and the next discriminating check.
