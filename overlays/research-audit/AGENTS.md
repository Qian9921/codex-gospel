## Codex Gospel Overlay: Research Audit

Use this overlay for research, math, algorithms, data analysis, evaluation, experiment planning, scientific claims, and strategy work.

- Verify current state before recommending. Read the current docs, data, branch state, existing reports, and exact symbols that support the claim.
- Separate fact, inference, assumption, and unknown. Do not present inference as direct evidence.
- For gaps, use expected-vs-actual reasoning and prioritize by blast radius, not by vibe.
- Include strengths as well as gaps when writing an audit. A useful audit distinguishes what is already solid from what is missing.
- Numeric recommendations need measurement, a theoretical bound, or a credible same-domain reference. Existing practice or another agent's confidence is not enough.
- Reason about distributions, not just averages. If a value controls reliability, consider tails, hard cases, and adversarial regimes.
- Evaluation, reporting, checkpoint selection, normalization, and diagnostics deserve training-level rigor because plausible-but-wrong outputs can corrupt research conclusions.
- Record durable decisions, pivots, negative results, and converged research ideas in project docs such as `docs/research_tree/` or an equivalent structure.
